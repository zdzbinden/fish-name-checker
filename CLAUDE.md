# Fish Name Checker — Project Guide

## What this project is

A static web app + PDF parser that validates scientific fish names in manuscript
text against *Common and Scientific Names of Fishes from the United States,
Canada, and Mexico*, 8th edition (Page et al., 2023 — AFS/ASIH).

Intended audience: fisheries scientists and ichthyologists proofreading manuscripts.
Goal: eventually host on GitHub Pages for public use by the field.

## PDF parsing

Run with:
```powershell
uv run --with pymupdf python parse_pdf.py
```

### Key PDF quirk

The book is landscape but stored as portrait in the PDF. Each book column
maps to a vertical **x-strip** (words sharing similar x0 coordinates).
Reading order = ascending x0 (left→right book column) and descending y0
within each strip (top→bottom of book entry).

Characters were historically stored in reverse stream order, but **PyMuPDF
(fitz) automatically returns text in correct visual order** and decomposes
ligatures. Do NOT reverse words or apply manual ligature patches — this was
needed with pdfplumber but PyMuPDF handles it.

### Page ranges (0-indexed)

| Section  | Start | End  | Book pages |
|----------|-------|------|------------|
| Species  | 34    | 229  | 35–230     |
| Appendix | 230   | 269  | 231–270    |

- ~5,335 valid species
- ~1,627 unique genera
- Synonyms: 0 (appendix parsing not yet implemented)

## Web app

Pure HTML + CSS + vanilla JS. No build step, no framework.
Loads `data/fish_names.json` via `fetch()` — must be served over HTTP.

```powershell
cd fish-name-checker
python -m http.server 8080
# open http://localhost:8080
```

Classification logic (`js/app.js`):
1. **Valid** (green) — exact match in 8th edition
2. **Outdated** (orange) — known synonym; suggests current name
3. **Misspelled** (red) — Levenshtein distance ≤ 2; suggests closest match
4. **Unknown** (purple) — genus looks fish-like but no close species found
