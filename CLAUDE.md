# FISHFINDER — Project Guide

## What this project is

**FISHFINDER** — a static web app + data pipeline that validates scientific fish
names in manuscript text against *Common and Scientific Names of Fishes from the
United States, Canada, and Mexico*, 8th edition (Page et al., 2023 — AFS/ASIH).

Design inspired by the classic Lowrance fish finder (retro green LCD aesthetic).

Intended audience: fisheries scientists and ichthyologists proofreading manuscripts.
Live at: https://zdzbinden.github.io/FISHFINDER/
GitHub: https://github.com/zdzbinden/FISHFINDER

## Data pipeline

### Step 1 — parse_pdf.py

```powershell
uv run --with pymupdf python parse_pdf.py
```

Parses `names_of_fishes/Names-of-Fishes-8-Table1.pdf` (183 pages, properly
oriented, clean text encoding — NOT the full book PDF).

**Format:** Each species row uses dot-leader column separators:
```
[flags\t]Genus species Author, Year .... OCC .... English .... Spanish .... French
```

**Flags:** `*` = changed from 7th edition, `^` = added via EEZ expansion

**Output — JSON fields per species:**
```json
{
  "Genus species": {
    "class":          "Actinopteri",
    "order":          "Perciformes",
    "family":         "Percidae",
    "author":         "(Meek, 1891)",
    "occurrence":     "F:U",
    "flags":          "*",
    "common_name_en": "Yoke Darter",
    "common_name_es": "",
    "common_name_fr": ""
  }
}
```

**Current output:** ~5,086 species, ~1,493 genera

**Known issue:** `order` field carries over incorrectly across page boundaries in
some cases (cosmetic; does not affect name checking).

### Step 2 — scrape_eschmeyer.py

```powershell
uv run --with requests --with beautifulsoup4 python scrape_eschmeyer.py
```

Queries Eschmeyer's Catalog of Fishes for each AFS species. Captures three types
of synonym relationships:

1. **Strict synonyms** — entries whose current status is "Synonym of \<target\>"
2. **Genus transfers / reclassifications** — entries listed as "Valid as \<target\>"
   but originally described under a different genus
3. **Historical synonym chains** — within a synonym entry, older "Synonym of X"
   or "Valid as X" references that name a different binomial for the same species

**HTML parsing notes:**
- `get_text(separator=" ")` with whitespace normalization (`\s+` → ` `, `\s+,` → `,`)
  to handle tag-boundary spaces from italicized names
- Entry headers matched with `epithet, Genus(?= Author [Initials])` lookahead to
  avoid false-matching geographic text (e.g., "Bay, California")
- "Look backwards" from each status occurrence to the nearest preceding entry header

**Safety filters:**
- Synonyms that are themselves valid AFS names are excluded from `fish_names.json`
  (prevents false positives from historical bullets that were later reinstated)
- AFS is treated as the authority for validity; Eschmeyer's `valid`/`current_name`
  fields are diagnostic only

Results cached to `eschmeyer_cache.json` — runs resume automatically if interrupted.
Full run takes ~3 hours. Run rate: 2s/request + 15s pause every 50 requests.

## Web app

Pure HTML + CSS + vanilla JS. No build step, no framework.
Loads `data/fish_names.json` via `fetch()` — must be served over HTTP.

```powershell
cd fishfinder
python -m http.server 8080
# open http://localhost:8080
```

### Architecture

Classification logic lives in `js/engine.js` — a pure module with no DOM
dependencies. It exports functions via `FishEngine` (browser global) or
`module.exports` (Node.js). `js/app.js` handles UI, Firebase, animations, and
event wiring — it calls engine functions via `FishEngine.*`.

### Classification logic (js/engine.js)

1. **Valid** (green) — exact match in 8th edition; hover shows common name
2. **Changed** (blue) — valid in 8th edition but `*`-flagged (changed from 7th);
   issues table shows current common name and prompts user to confirm species identity.
   Critical for cases like *Micropterus salmoides* (now Florida Bass, not Largemouth Bass)
3. **Outdated** (orange) — known synonym; suggestion shows corrected name + common name
4. **Misspelled** (red) — Levenshtein distance ≤ 2; shows closest match
5. **Unknown** (purple) — genus looks fish-like but no close species found

### Key taxonomy note

In the 8th edition, Largemouth Bass changed from *Micropterus salmoides* to
*Micropterus nigricans*. *Micropterus salmoides* now refers to Florida Bass only.
Both carry the `*` flag and will appear as "Changed in 8th edition."

## Testing

```powershell
cd fishfinder
node --test test/*.test.js
```

Uses Node.js built-in test runner (`node:test` + `node:assert`). Zero npm
dependencies. Tests load `fish_names.json` directly via `fs.readFileSync` and
exercise the engine against the real dataset.

**Test files:**
- `levenshtein.test.js` — edit-distance algorithm unit tests
- `classify.test.js` — classification decision tree (valid, changed, outdated,
  misspelled, unknown, common name, abbreviation filtering)
- `extract.test.js` — text extraction (binomial regex + common name matching)
- `edge-cases.test.js` — fuzzy matching boundaries, charCode filters, database
  integrity checks

Add new test cases here to iteratively discover improvements to the engine.

## Design decisions

- **AFS is the authority.** Eschmeyer is used only for synonym discovery. If
  Eschmeyer considers an AFS-valid name to be a synonym, that's logged as a
  mismatch but doesn't change the checker's behavior.
- **False-positive safety net.** Any synonym that is itself a valid AFS name is
  filtered out before writing to `fish_names.json`. This handles cases where a
  historical "Synonym of X" bullet was later reinstated as valid.

## Engine development roadmap

Informed by a literature review of taxonomic screening pipelines (Rees 2014,
Zermoglio et al. 2016, Le Guillarme & Thuiller 2022, Leone et al. 2026,
Grenié et al. 2023). Full audit in `publication/literature/`.

### Benchmarking

- **Zermoglio et al. (2016) benchmark** — Their publicly available dataset of
  1,000 human-vetted verbatim vertebrate names from VertNet is a gold-standard
  test set. Filter to fish records, run through FISHFINDER engine, measure
  precision/recall/F1 per classification tier. Create `test/benchmark-zermoglio.test.js`.
- **Synthetic misspelling test set** (Rees 2014 methodology) — Programmatically
  corrupt known valid fish names with controlled edit operations (substitution,
  deletion, insertion, transposition) at edit distances 1 and 2. Measure
  recall/precision and compare against Taxamatch's published figures.
- **TaxoNERD comparison** (Le Guillarme & Thuiller 2022) — Run TaxoNERD on
  10–20 fisheries manuscripts alongside FISHFINDER's `extractCandidates`.
  Compare detection rates; TaxoNERD will likely find abbreviated genera
  (*M. salmoides*) that FISHFINDER's regex misses.
- **Cross-database synonym validation** (Leone et al. 2026 / rfishbase) — Run
  TaxonMatch or rfishbase on AFS species, diff against `fish_names.json` to
  find synonym coverage gaps and AFS-vs-GBIF authority divergences.

### Algorithm improvements

- **Damerau-Levenshtein distance** (Rees 2014) — Replace standard Levenshtein
  with Optimal String Alignment variant so transpositions (e.g., "Cyrpinus" →
  "Cyprinus") cost 1 edit instead of 2. Minimal change: add transposition
  check + `prevprev` row in `levenshtein()`. Low effort, medium impact.
- **Phonetic pre-screening fallback** (Rees 2014) — Lightweight Latin/Greek
  phonetic normalization (ph→f, ch→k, th→t, ae→e, double consonants collapsed).
  Use as fallback when standard fuzzy match returns nothing at distance ≤ 2.
  Would catch "Oncorhynkus" → "Oncorhynchus" type errors. Medium effort.
- **Confidence scoring** (Leone 2026, Grenié 2023) — Extend `classifyName`
  return with `confidence` and `editDistance` fields (exact=1.0, synonym=0.95,
  fuzzy synonym=0.80, misspelled d=1→0.70, d=2→0.50). Low effort.
- **Context-aware false positive filtering** (Le Guillarme 2022) — Expand
  genus-preceding word checks ("University", "Journal", "Department", etc.)
  as lightweight heuristic NER approximation. Low effort.

## Open issues / future work

- `family` (and `order`/`class`) fields carry over incorrectly across page
  boundaries in `parse_pdf.py`, affecting 360 species (7%) across 54 family
  pairs (e.g., Gobiesocidae species assigned to Mugilidae). Cosmetic only —
  these fields are not used by the classification engine, synonyms, or UI.
  Fix requires re-running the parser with page-boundary-aware family tracking.
- Species count: Table 1 contains exactly 5,086 species (confirmed from both
  the table-only and full book PDFs). The Introduction states 5,089 — this is
  an internal inconsistency in the book, not a parser bug.
- `og:image` meta tag needs a 1200x630 preview card (graphic design task)
- Companion paper for *Fisheries* in progress (publication/ folder)
