## Project Goal

Build a **Fish Name Checker** — a tool that lets ichthyologists paste their manuscript text, automatically detects scientific (Latin binomial) fish names, checks them for **spelling errors** and **outdated nomenclature** against the authoritative reference "Common and Scientific Names of Fishes from the United States, Canada, and Mexico" (8th edition, AFS/ASIH, 2023), and reports issues with suggested corrections.

Deliver two things:

1. **A pre-processed JSON data file** extracted from the PDF (the "dictionary")

## Step 1: Parse the PDF and Build the Data File

The reference PDF is at `./names_of_fishes/NAMES OF FISHES 8th.pdf` (449 pages).

### PDF structure you need to know

- **Pages ~1–34**: Front matter, introduction, list of families (table of contents for families)
- **Pages ~35–230**: **The main species list** — this is the primary data source
- **Pages ~231–270**: **Appendix 1 — "Changes from Seventh Edition (2013)"** — contains old→new name mappings
- **Pages ~271–449**: Index (alphabetical lookup of all names with page refs)

### Main species list format (pages ~35–230)

Each page has entries like this:

```
Sphyrna lewini (Griffith & Smith, 1834)........A-P:MU.........Scalloped Hammerhead........cornuda común
```

The columns are:
- **Scientific name**: `Genus species` followed by author(s) and year — e.g., `Sphyrna lewini (Griffith & Smith, 1834)`
- **Occurrence codes**: geographic/habitat codes like `A-P:MU`, `F:CMU`, etc.
- **Common names**: English name, then optionally Spanish and/or French

Entries are grouped under **ORDER** headings (e.g., `ORDER SQUALIFORMES`) and **Family** headings (e.g., `Sphyrnidae–En-hammerhead sharks, Sp-tiburones martillo, Fr-requins marteaux`).

Symbols before entries:
- `*` = changed from the 7th edition
- `^` = newly added (EEZ expansion species)

### Appendix 1 format (pages ~231–270)

Free-text paragraphs organized by page number of the main list. Contains entries like:
- `"Kyphosus sectatrix. New to the list. Replaces K. saltatrix in previous editions"`
- `"Hyporthodus cifuentesi. Transferred from Epinephelus"`
- `"Etheostoma occidentale... removed from the list based on synonymization with E. atripinne"`
- `"Masturus lanceolatus. Transferred from Mola"`

### What to extract → `fish_names.json`

Use Python with `pdfplumber` (preferred for layout-aware extraction) or `pdftotext` (CLI) to extract:

```json
{
  "metadata": {
    "edition": 8,
    "year": 2023,
    "source": "Common and Scientific Names of Fishes (AFS/ASIH)",
    "species_count": 5089
  },
  "valid_names": {
    "Sphyrna lewini": {
      "family": "Sphyrnidae",
      "order": "Carcharhiniformes",
      "common_name_en": "Scalloped Hammerhead",
      "occurrence": "A-P:MU"
    }
  },
  "genera": ["Sphyrna", "Carcharhinus", ...],
  "synonyms": {
    "Kyphosus saltatrix": "Kyphosus sectatrix",
    "Epinephelus cifuentesi": "Hyporthodus cifuentesi",
    "Mola lanceolatus": "Masturus lanceolatus",
    "Etheostoma occidentale": "Etheostoma atripinne"
  }
}
```

**Parsing guidance:**

1. Extract text from pages 35–230 using layout-preserving mode (`pdftotext -layout` or `pdfplumber`).
2. For each line, detect scientific names using a regex pattern: look for lines starting with optional `*` or `^`, then a capitalized word (genus) followed by a lowercase word (species), followed by author/year in parentheses or not. The genus+species binomial is the key.
3. Track the current ORDER and Family heading as context.
4. For Appendix 1 (pages ~231–270): parse "Transferred from [OldGenus]" and "Replaces [OldName]" and "synonymization with [NewName]" patterns to build the `synonyms` map. This does NOT need to be perfect — capture what you can with reasonable regex/NLP. Some will be missed and that's fine.
5. Also extract every unique genus name into the `genera` array (used for detecting whether a word in user text is likely a fish genus).

**Write the parser as `parse_pdf.py`** so it can be re-run when a new edition is published. Output `fish_names.json`.

## Step 2: Build the Static Web App

Create a single-page app with **HTML + CSS + vanilla JavaScript** (no frameworks, no build step, no server). The entire app lives in one folder deployable to GitHub Pages.

### File structure

```
fish-name-checker/
├── index.html          # The app
├── css/
│   └── style.css
├── js/
│   └── app.js          # All logic
├── data/
│   └── fish_names.json # Generated by parse_pdf.py
├── parse_pdf.py        # PDF parser (not deployed, just in repo)
└── README.md           # Instructions for use and updating
```

### How the checker works (app.js logic)

1. **On page load**: fetch `data/fish_names.json` and build lookup structures:
   - A `Set` of all valid binomials (lowercased for matching)
   - A `Map` from old/synonym names to current valid names
   - A `Set` of all known genera

2. **User pastes or types text** into a large textarea.

3. **On clicking "Check Names"**: scan the text for candidate scientific names using this approach:
   - Find all instances of a capitalized word followed by one or more lowercase Latin words: regex pattern like `/\b([A-Z][a-z]{2,})\s+([a-z]{2,})\b/g`
   - Filter candidates: keep only those where the genus matches a known genus from the data, OR where the genus is within edit distance ≤2 of a known genus (to catch misspelled genera)
   - This avoids false positives from ordinary English phrases

4. **For each candidate name found, classify it:**
   - **Valid**: exact match in `valid_names` → highlight green
   - **Outdated/Synonym**: exact match in `synonyms` → highlight orange, show the current valid replacement
   - **Misspelled**: no exact match, but within Levenshtein distance ≤2 of a valid name → highlight red, suggest the closest match(es)
   - **Unknown**: looks like a scientific name (genus is close to a known genus) but no close match → highlight red, flag for manual review

5. **Display results** in a results panel below the textarea:
   - Show the original text with inline colored highlights on flagged names
   - Show a summary table listing each flagged name, the issue type, and the suggested correction
   - Allow copying the corrected text

### Levenshtein / fuzzy matching

Implement a simple Levenshtein distance function in JS. For performance with ~5,000 species:
- First check exact match (O(1) via Set)
- Then check synonyms (O(1) via Map)
- Only compute edit distance if not found — and limit candidates by matching the first letter of genus + first letter of species to reduce comparisons
- Suggest matches with distance ≤2

### UI Design

Keep it minimal and professional:
- Clean sans-serif font (system font stack)
- A header with the tool name: **"Fish Name Checker"** and subtitle: *"Validate scientific names against Names of Fishes, 8th edition (AFS/ASIH, 2023)"*
- A large textarea (at least 20 rows) with placeholder text: *"Paste your manuscript text here..."*
- A "Check Names" button
- Results area with:
  - Color-coded legend (green = valid, orange = outdated, red = misspelled/unknown)
  - Highlighted text preview
  - Summary table with columns: Found Name | Issue | Suggested Correction
- A footer with a note: *"Based on Common and Scientific Names of Fishes from the United States, Canada, and Mexico, 8th edition (Page et al., 2023)"*
- Make it responsive (works on mobile)

### Important implementation details

- The JSON file will be ~1–2MB. Load it asynchronously on page load and show a loading indicator.
- All processing happens client-side in the browser. No server calls.
- Use Web Workers if fuzzy matching is slow on large texts, but try the simple approach first.
- The app should handle texts up to ~50,000 words (a full manuscript).
